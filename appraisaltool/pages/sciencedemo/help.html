Frequently Asked Questions


What is the purpose of the technical appraisal tool?

The tool assists the user in assessing the proximity and impact of external risks to complex digital objects. Currently


What is a complex digital object?

A complex digital object is made up of interdependent simple digital objects or components (e.g. files, software applications, hardware specifications).


What types of complex digital objects can be appraised?

Currently the tool supports digital video artworks (DVAs) and space science experiments.

A DVA comprises a digital video and its playback environment. More precisely, this comprises a video file (compressed with a video codec and stored in a container or wrapper), a media player, an operating system and a computer.

A science experiment comprises a database for raw data, a script written in a specific scripting language, images and documents in specific formats together with associated viewers.

A visual representation of the object structures can be viewed by clicking on any object from the Collection View.


What types of risks can be assessed?

The main risk that occurs is availability of the components of the complex digital object. This can be caused through obsolescence of the components or failure of the associated hardware (e.g. computers).


What information is provided about the risks?

Besides the type of the risk, the tool provides

  • Date for Action: this is the expected date at which the user is recommended to take action to mitigate for the primary risk to the object. This might involve replacing one or more of the components.
  • Confidence: the statistical confidence interval associated to the date for action. This provides the user with a date range in which they are recommended to perform mitigating actions on the object to ensure ongoing reusability.
  • Impact: this has three levels which are determined based on the cost of the least expensive mitigating actions.

What is the structure of the tool?

The tool has separate areas form media and science. This can be selected on the landing page or from Start on the top menu bar.The UI is divided into three main parts.

  • Collection View shows all the objects represented in the system together with their associated risks.
  • Component View can be selected from the top menu bar and shows all the specific technologies used in the complex digital objects and their associated individual risks.
  • Object View can be selected by clicking on any object in the collection view.

Where are the digital objects stored?

The technical appraisal tool stores representations of the objects in a triple store (either the PERICLES Entity Registry Model Repository) or Fuseki. The objects themselves need to be stored in an external system, which could be an archive or repository.


What data sources are used?

The tool harvests and analyses data automatically from search engines, software repositories and Wikipedia.


What ontologies are used?

The tool uses inbuilt digital ecosystem models for the media and science domains. These determine the component technologies used in the objects, the compatibility between those components and the transformation costs for replacing components. The models can be edited using the separate PERICLES-IBuilder application, but not currently through the technical appraisal tool UI.


What are "recovery options"?

A recovery option is a replacement of one or more components in an artwork. The objective of a recovery option is to move forward the date for action by replacing components at risk of obsolescence or failure, and thus extend the potential lifetime of the object. There are often multiple options depending on how far the date for action is moved forward. The intended retention period of the object will be a factor in selecting an appropriate recovery option. Also, the more changes that are made to the object, the more work that is required and hence the cost will be higher. The tool selects the optimal recovery options for a given object. Recovery options may involve transcoding video or image files, rewriting scripts or replacing hardware. If no recovery options are provided, then no actions are currently recommended.
